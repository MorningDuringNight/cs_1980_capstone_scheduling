Packaging Instructions

This application can be packaged into a native installer using `jpackage`. The application JAR must be built before packaging, since `jpackage` uses the compiled JAR as input.

Build the project first:

mvn clean package (must be in capstone-scheduling directory)

After the build completes, run `jpackage` from the project root. Example:

jpackage --input target/ \
  --name CapstoneScheduling \
  --main-jar capstone-scheduling-1.0-SNAPSHOT.jar \
  --main-class capstoneSchedulingApp.App \
  --type dmg

Arguments:

--input: directory containing the built JAR  
--name: name of the packaged application  
--main-jar: JAR file to launch  
--main-class: application entry point  
--type: installer format for the current operating system  

Supported package types depend on the OS:
- macOS: dmg, pkg
- Windows: exe, msi
- Linux: deb, rpm

Examples:
- macOS: use `--type dmg`
- Windows: use `--type exe`
- Ubuntu/Linux: use `--type deb`

The generated installer can then be distributed and installed like a normal desktop application.
